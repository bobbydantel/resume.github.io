Юридические AI-консультанты

Контекст

Компания хотела создать юридического помощника на ИИ, который давал бы рекомендации по рекламным публикациям и предупреждал о возможных рисках. Был готов прототип, который необходимо было улучшить.

Задачи

  • Увеличить точность ответов
  • Добавить ссылки на правовые акты
  • Уменьшить количество галлюцинаций

Действия

В первую очередь я написал промпт, который уменьшал количество галлюцинаций. Для повышения точности ответов был собран сценарий в n8n, который использует векторное хранилище с правовым актом, что позволило агенту ссылаться на закон, а не придумывать их.

Результат

  • ✔ Точность ответов выросла до 90%
  • ✔ Агент стал безошибочно ссылаться на прововые акты
  • ✔ Галлюцинации получилось исключить полностью

Автоматизация поиска контента

Контекст

Редактор в компании тратил от 3 часов ежедневно на поиск контента для корпоративных и клиентских каналов.

Задача

Сокращение времени поиска контента.

Действия

Был собран сценарий в n8n, который искал новости по RSS-фидам. Далее агент саммаризировал контент по найденным источникам и отправлял его в базу данных. Далее второй агент в режиме чата позволял искать необходимый контент по ключевым словам.

Результат

До

20 инфоповодов / ~3 часа

После

650 инфоповодов / 4-5 минут

Стоимость задачи поиска контента снизилась на 90%.

Агент технической поддержки

Ситуация

Отдел тех.поддержки тратил много времени на обработку однообразных заявок, разрешение которых находилось в документации сервиса. Пользователи не изучали документацию самостоятельно и для решения всех базовых проблем отправляли заявки. У техподдержки уходило около суток на решение проблемы.

Задача

Автоматизировать первый уровень обработки заявок и снизить время отклика для решения проблем пользователей.

Действия

  • Совместно с командой разработки был создан MCP клиент, который позволял получать информацию напрямую из профиля пользователя.
  • В n8n был собран агент, который позволял получать данные с MCP, обрабатывать их и давать первичную оценку конфигураций из профиля клиента и давать рекомендации по устранению ошибок.
  • Был подключен второй уровень обработки с RAG, по которому пользователь может задать вопрос по своей проблеме и получить ответ на основе документации сервиса.

Результат

Время решения базовых проблем с настройками конфигураций было снижено с рабочих суток до 1-2 минут.

Сложные проблемы пользователя, которые требовали вмешательства техподдержки, не терялись в потоке других базовых ошибок.
